## AC (인수조건) 작성 패턴

### 기본 원칙
- **측정 가능**: 숫자/시간/횟수로 검증 가능해야 한다
- **조건-행동 구조**: "~할 때 ~한다" 형식
- **독립적**: 다른 FR 없이도 개별 검증 가능

### 기능 유형별 AC 패턴

#### 목록/필터 기능
```
- 필터 {N}개 조합 시 결과 {N}초 이내 표시
- 결과 0건 시 빈 상태 안내 + 필터 초기화 버튼 표시
- URL 파라미터로 필터 상태 유지 (뒤로가기 시 복원)
- 페이지당 {N}건 표시, 총 건수 표시
```

#### 상세 페이지
```
- 필수 정보 항목 전체 렌더링 (정보 없는 항목은 "정보 없음" 표시)
- 이미지 갤러리: 대표 이미지 + 서브 이미지, 라이트박스 확대
- 공유 버튼 클릭 시 현재 페이지 URL + OG 메타 공유
- 관련 콘텐츠 {N}건 자동 추천
```

#### 폼/문의
```
- 필수 입력값 미입력 시 해당 필드 하이라이트 + 에러 메시지
- 이메일/연락처 형식 유효성 검사 (클라이언트 + 서버)
- 제출 완료 시 접수번호 표시 + 담당자 이메일 알림
- 파일 첨부: 확장자 화이트리스트, 용량 제한 {N}MB
```

#### 검색
```
- 최소 {N}자 입력 후 검색 실행
- 자동완성: 입력 {N}ms 디바운스
- 검색 결과 키워드 하이라이트
- 결과 0건 시 추천 키워드 표시
```

#### 결제/트랜잭션
```
- 결제 완료까지 {N}단계 이내
- 재고 최종 확인 (결제 직전)
- 결제 실패 시 재시도 안내 + 재고 복원
- 결제 성공 시 주문 확인 알림 (이메일/알림톡)
```

### AC 안티패턴

| 안티패턴 | 문제 | 개선 |
|---------|------|------|
| "빠르게 로딩" | 측정 불가 | "1초 이내 로딩" |
| "사용하기 쉬움" | 주관적 | "3클릭 이내 목표 도달" |
| "보기 좋게" | 검증 불가 | "WCAG AA 명도 대비 충족" |
| "안정적 운영" | 모호 | "가용성 99.5%, 응답 3초 이내" |

## NFR 표준 측정기준

### 성능 (NFR-001)
| 지표 | 기준 | 측정 도구 |
|------|------|----------|
| LCP | ≤ 2.5초 | Lighthouse |
| FCP | ≤ 1.8초 | Lighthouse |
| CLS | ≤ 0.1 | Lighthouse |
| 전체 점수 | ≥ 90 (모바일) | Lighthouse |

### 반응형 (NFR-002)
| 브레이크포인트 | 범위 |
|--------------|------|
| 모바일 | ≤ 768px |
| 태블릿 | 769 ~ 1024px |
| 데스크톱 | ≥ 1025px |

### 접근성 (NFR-003)
| 항목 | 기준 |
|------|------|
| WCAG 레벨 | 2.1 AA |
| 검사 도구 | axe, WAVE |
| 합격 기준 | Critical/Serious 이슈 0건 |

### SEO (NFR-004)
| 항목 | 필수 |
|------|------|
| meta title/description | 전 페이지 |
| Open Graph 태그 | 전 페이지 |
| Schema.org 마크업 | 주요 콘텐츠 페이지 |
| sitemap.xml | 필수 |
| robots.txt | 필수 |

### 보안 (NFR-005)
| 항목 | 기준 |
|------|------|
| SSL | HTTPS 전체 적용, 리다이렉트 |
| XSS 방지 | 입력값 sanitize |
| CSRF 방지 | 폼 토큰 적용 |
