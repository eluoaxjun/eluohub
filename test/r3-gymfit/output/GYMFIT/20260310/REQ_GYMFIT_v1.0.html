Functional Requirements
기능 요구사항 목록 (FR-001 ~ FR-023)
A. 인증 / 계정 관리
FR-001Must
운영자 계정 가입
헬스장 운영자가 이메일·비밀번호로 플랫폼에 가입한다
  • 가입 완료 시 JWT 토큰이 발급된다
  • 중복 이메일로 가입 시 "이미 사용 중인 이메일" 오류가 표시된다
  • 가입 완료 시 카카오 알림톡으로 환영 메시지가 발송된다
QST 근거: PROJECT.md 기확정
FR-002Must
사용자 역할 권한 관리
운영자·강사·회원 3종 역할을 구분하고 역할별 접근 권한을 제어한다
  • 운영자는 강사·회원 등록·수정·삭제가 가능하다
  • 강사는 본인 수업 일정 조회·관리만 가능하다
  • 회원은 예약·조회·취소만 가능하다
  • 역할 없는 사용자는 로그인 후 권한 없음 안내가 표시된다
QST 근거: Q-004 가정: 3종
FR-003Should
강사·스태프 초대
운영자가 이메일 초대 링크로 강사·스태프 계정을 생성한다
  • 초대 링크는 발송 후 24시간 내 유효하다
  • 이미 가입된 이메일로 초대 시 "이미 가입된 사용자" 안내가 표시된다
  • 초대 수락 완료 시 운영자에게 알림이 발송된다
QST 근거: Q-006 가정: 이메일 초대
B. 수업 예약 시스템
FR-004Must
그룹 수업 예약
회원이 수업 목록에서 예약 가능한 그룹 수업을 선택·예약한다
  • 정원 미만 수업만 예약 버튼이 활성화된다
  • 예약 완료 시 카카오 알림톡 확인 메시지가 발송된다
  • 동일 수업 중복 예약 시 "이미 예약된 수업" 안내가 표시된다
QST 근거: Q-007 가정: A+B 그룹+PT
FR-005Must
PT 1:1 세션 예약
회원이 강사·날짜·시간을 선택하여 PT 세션을 예약한다
  • 강사 휴가 등록 시간대는 예약 불가 상태로 표시된다
  • 예약 완료 시 강사·회원 양쪽에 카카오 알림톡이 발송된다
  • 회원당 동시 PT 예약은 최대 5건까지 허용된다
QST 근거: Q-007 가정: A+B 그룹+PT
FR-006Must
예약 취소·변경
회원이 예약을 취소하고 운영자가 예약을 강제 취소할 수 있다
  • 수업 시작 2시간 전까지 회원이 직접 취소 가능하다
  • 취소 완료 시 카카오 알림톡으로 취소 확인 메시지가 발송된다
  • 운영자는 취소 기한과 무관하게 강제 취소 가능하다
QST 근거: Q-008 미확인
FR-007Should
대기자 명단 관리
정원 초과 수업에 대기 등록하고, 취소 발생 시 자동 예약 전환한다
  • 정원 초과 수업에 대기 등록 시 대기 순번이 부여된다
  • 취소 발생 시 1번 대기자에게 자동 전환 알림이 발송된다
  • 대기자는 알림 수신 후 1시간 내 확인하지 않으면 다음 대기자에게 넘어간다
QST 근거: Q-009 가정: 자동 전환
C. 수업·강사 일정 관리
FR-008Must
수업 일정 등록·관리
운영자가 수업명·강사·시간·정원을 설정하여 수업을 등록한다
  • 반복 수업(매주 특정 요일)을 일괄 등록할 수 있다
  • 수업 수정·삭제 시 이미 예약한 회원에게 자동 알림이 발송된다
  • 오늘 이전 날짜의 수업은 수정 불가 처리된다
QST 근거: PROJECT.md 목적
FR-009Must
강사 스케줄 관리
강사가 본인 수업 일정과 부재 시간대를 관리한다
  • 강사가 캘린더 뷰에서 본인 수업 일정을 조회할 수 있다
  • 강사가 부재 시간대를 등록하면 해당 시간대 PT 예약이 자동 차단된다
  • 강사 부재 시간대가 기존 예약과 충돌 시 운영자에게 알림이 발송된다
QST 근거: PROJECT.md 목적
D. 회원 관리
FR-010Must
회원 등록·관리
운영자가 회원 기본 정보를 등록하고 관리한다
  • 이름·연락처·회원권 유형으로 회원을 등록할 수 있다
  • 회원 목록에서 이름·연락처로 검색이 가능하다
  • 회원 정보 수정 시 변경 이력이 기록된다
QST 근거: PROJECT.md 목적
FR-011Must
회원권 관리
월정액·횟수권·기간권 등 다양한 회원권 유형을 설정하고 관리한다
  • 운영자가 3가지 이상 회원권 유형을 생성할 수 있다
  • 회원권 만료 7일 전에 자동 리마인더 알림이 발송된다
  • 만료 회원권 보유 회원의 예약 시도 시 "회원권 만료" 안내 후 결제 페이지로 이동한다
QST 근거: Q-010 가정: 월정액/횟수/기간 3종
FR-012Must
회원 출석 체크
운영자·강사가 수업별 출석을 기록한다
  • 수업 당일 회원별 출석·결석·지각을 선택하여 기록할 수 있다
  • 출석 기록은 회원 이력 페이지에서 날짜 범위로 조회 가능하다
  • 횟수권 회원의 출석 시 잔여 횟수가 1 차감된다
QST 근거: PROJECT.md 목적
FR-013Should
회원 이력 조회
운영자가 특정 회원의 예약·결제·출석 통합 이력을 조회한다
  • 회원 상세 페이지에서 날짜 범위로 이력을 조회할 수 있다
  • 이력 데이터를 CSV로 내보낼 수 있다
  • 최근 30일 이력이 기본값으로 표시된다
QST 근거: PROJECT.md 목적
E. 결제 관리
FR-014Must
포트원 온라인 결제
포트원 PG를 통해 회원권 온라인 결제를 처리한다
  • 신용카드·간편결제(카카오페이, 토스페이)로 결제가 가능하다
  • 결제 실패 시 "결제 실패" 안내와 재시도 버튼이 표시된다
  • 결제 완료 시 영수증이 카카오 알림톡으로 자동 발송된다
QST 근거: PROJECT.md 필수 연동
FR-015Must
오프라인 결제 기록
운영자가 현금·현장 결제를 수동으로 기록한다
  • 운영자가 결제 수단(현금/카드단말기)·금액·회원권 유형을 입력하여 결제 기록을 생성할 수 있다
  • 오프라인 결제 기록이 매출 리포트에 포함된다
  • 수정·삭제 시 이력이 남는다
QST 근거: Q-011 가정: 온오프혼합
FR-016Should
정산·매출 관리
운영자가 기간별 매출 현황과 강사별 수입을 조회한다
  • 일·주·월·년 기준 매출 합계와 결제 건수를 조회할 수 있다
  • 회원권 유형별 매출 분포 차트가 제공된다
  • 정산 리포트를 PDF·CSV로 내보낼 수 있다
QST 근거: Q-002
F. 알림·커뮤니케이션
FR-017Must
카카오 알림톡 자동 발송
예약·취소·리마인더·결제 이벤트에 카카오 알림톡을 자동 발송한다
  • 예약 생성·취소·변경 시 회원에게 카카오 알림톡이 3분 이내 발송된다
  • 알림톡 발송 이력이 관리자 페이지에서 조회 가능하다
  • 발송 실패 시 SMS로 대체 발송된다
QST 근거: PROJECT.md 필수 연동 / Q-012 가정: 카카오+SMS 대체
FR-018Must
예약 리마인더 발송
수업 전 일정 시간에 자동 리마인더 알림을 발송한다
  • 수업 24시간 전·1시간 전 자동 리마인더가 발송된다
  • 운영자가 리마인더 발송 시점(시간 단위)을 설정할 수 있다
  • 회원이 알림 수신 거부 시 리마인더가 발송되지 않는다
QST 근거: PROJECT.md 필수 연동
G. 대시보드·리포트
FR-019Must
운영자 메인 대시보드
운영자가 오늘의 예약 현황·회원 수·매출 합계를 한눈에 확인한다
  • 로그인 후 메인 화면에 오늘 예약 수, 총 회원 수, 이번달 매출이 카드 형태로 표시된다
  • 최근 7일·30일 예약 트렌드 차트가 제공된다
  • 대시보드 데이터는 페이지 진입 시 자동 갱신된다
QST 근거: PROJECT.md 기대 효과
FR-020Must
예약 현황 조회
운영자·강사가 날짜별·수업별 예약 현황을 조회한다
  • 캘린더 뷰와 리스트 뷰 전환이 가능하다
  • 수업별 예약자 명단을 PDF로 인쇄할 수 있다
  • 특정 회원의 예약 이력을 검색으로 조회할 수 있다
QST 근거: PROJECT.md 목적
H. SaaS 구독 관리
FR-021Must
헬스장 구독 플랜 관리
운영자가 구독 플랜을 선택하고 플랫폼 이용 요금을 관리한다
  • 운영자 가입 시 구독 플랜 선택 화면이 제공된다
  • 구독 만료 7일 전 자동 갱신 안내 알림이 발송된다
  • 구독 만료 후 회원 예약 기능이 비활성화된다
QST 근거: Q-001 가정: 월 구독 정액제
I. 랜딩·온보딩
FR-022Should
서비스 랜딩 페이지
헬스장 운영자 대상 서비스 소개와 무료 체험 신청 CTA를 제공한다
  • 주요 기능·요금제·고객 후기 섹션이 포함된다
  • "무료 체험 시작" CTA 클릭 시 가입 페이지로 이동한다
  • 모바일·데스크톱 반응형으로 제공된다
QST 근거: Q-003 가정: 무료 체험 제공
FR-023Should
데이터 내보내기
운영자가 회원·예약·매출 데이터를 CSV로 내보낼 수 있다
  • 회원 목록·예약 이력·매출 데이터를 CSV(UTF-8)로 내보낼 수 있다
  • 내보내기 시 날짜 범위 필터를 적용할 수 있다
  • 내보내기 완료 시 다운로드가 자동 시작된다
QST 근거: Q-013 가정: CSV export만
Non-Functional Requirements
비기능 요구사항 (NFR-001 ~ NFR-007)
NFR-001
성능
핵심 페이지 로딩 속도 — 예약 목록, 대시보드, 회원 관리 등 주요 화면
LCP ≤ 2.0초 / Lighthouse 모바일 ≥ 70점
NFR-002
반응형
모바일 퍼스트 반응형 레이아웃 — 운영자·강사·회원 모두 모바일에서 핵심 기능 사용 가능
320px / 768px / 1280px — Chrome/Safari DevTools 기준
NFR-003
접근성
웹 접근성 준수 — 색 대비, 키보드 내비게이션, 스크린리더 지원
WCAG 2.1 AA — 색 대비 4.5:1 이상, 키보드 내비게이션
NFR-004
보안
인증·데이터 암호화 — 회원 개인정보 및 결제 정보 보호
JWT 인증 / HTTPS 필수 / 개인정보(연락처·결제정보) AES-256 암호화
NFR-005
가용성
서비스 안정성 — 헬스장 운영 시간(06:00~23:00) 중 무중단 서비스
월간 가동률 99.9% (월 다운타임 43분 이내)
NFR-006
브라우저 호환성
크로스브라우저 지원 — 운영자·강사는 데스크톱 크롬/사파리, 회원은 모바일 환경
Chrome · Safari 최신 2버전 / iOS Safari · Android Chrome
