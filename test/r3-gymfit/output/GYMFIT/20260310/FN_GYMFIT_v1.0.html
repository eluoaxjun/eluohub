예외

기술 노트
Tech Note

기술 선택: JWT (Access + Refresh 이중 토큰), bcrypt 해시

대안: Session 기반 인증 (Redis 세션 스토어)

선택 사유: AWS EC2 수평 확장 시 stateless 필요 → JWT 선택. 세션은 스케일아웃 시 세션 공유 문제 발생

제약: Refresh Token 탈취 방지를 위해 httpOnly Cookie 필수, XSS 방어 병행

검증 기준
정상

예외

기술 노트
Tech Note

기술 선택: RBAC (Role-Based Access Control), JWT role 클레임

대안: ABAC (Attribute-Based), DB role 조회

선택 사유: 3종 고정 역할이므로 RBAC로 충분. DB 조회 없이 JWT에서 즉시 판별 가능

제약: 역할 변경 시 기존 JWT 즉시 무효화 필요 (Refresh Token 블랙리스트 관리)

기술 노트
Tech Note

기술 선택: Optimistic UI (예약 즉시 UI 반영) + 서버 확인 후 롤백 패턴

대안: Pessimistic UI (서버 응답 후 UI 업데이트)

선택 사유: 모바일 체감 속도 향상. 3클릭 예약 UX 목표 달성

제약: 동시 예약 race condition 방지를 위해 DB 트랜잭션 잠금 필수
