3. 기능 상세 정의

FN-001: 패키지 목록 조회

  • FR 근거: FR-001 (Must)
  • 기획 근거: 마켓플레이스 핵심 진입점. 카드 기반 목록으로 시각적 탐색을 유도하며, 3축 필터링(파이프라인/기능/업종)으로 사용자 탐색 효율을 극대화한다.
  • IA 근거: 생략 (IA 미생성)
  • 복잡도: 높음

화면 구성

영역 구성요소 설명
상단 카테고리 필터 바 3축 필터(파이프라인/기능/업종) 탭 또는 드롭다운
상단 우측 정렬 드롭다운 인기순/최신순/평점순
중앙 패키지 카드 그리드 12개씩 표시. 카드: 아이콘, 이름, 설명(2줄), 별점, 설치 수
하단 페이지네이션 또는 무한스크롤 (사용자 설정)

입력

항목 타입 필수 검증 규칙
카테고리 필터 select(multi) 아니오 3축 각각 0~N개 선택
정렬 기준 select 아니오 기본값: 인기순
페이지 번호 number 아니오 기본값: 1, 양수

처리

  • 정상 흐름:
    1. 페이지 진입 시 전체 패키지를 인기순으로 12개 조회
    2. 카테고리 필터 선택 시 해당 조건으로 재조회 (AND 조건)
    3. 정렬 변경 시 현재 필터 유지하며 재정렬
    4. 페이지네이션 클릭 또는 스크롤 시 다음 12개 로드
  • 예외 처리:
    • 결과 0건: "조건에 맞는 패키지가 없습니다. 필터를 조정하거나 검색해 보세요." + 검색 바 링크
    • 빈 카테고리: 카테고리 목록에서 자동 숨김
  • 에러 처리:
    • API 응답 실패: "목록을 불러올 수 없습니다. 잠시 후 다시 시도해 주세요." + 재시도 버튼
    • 타임아웃(3초 초과): 캐시된 이전 결과 표시 + "최신 정보가 아닐 수 있습니다" 안내

출력

항목 타입 설명
패키지 카드 component[] 아이콘, 이름, 요약(80자), 별점(평균), 설치 수, 카테고리 태그
총 건수 number 필터 적용 후 전체 패키지 수
페이지 정보 object 현재 페이지, 전체 페이지 수

검증 기준

구분 시나리오 기대 결과
정상 필터 없이 목록 진입 패키지가 인기순으로 12개씩 카드 표시
정상 카테고리 2개 선택 두 카테고리 모두 포함하는 패키지만 표시
정상 정렬을 최신순으로 변경 등록일 기준 내림차순 재정렬
예외 필터 결과 0건 빈 상태 메시지 + 검색 안내 표시
에러 API 500 에러 에러 메시지 + 재시도 버튼 표시
  • 비기능: NFR-001 (LCP 1.5초 이내), NFR-002 (API 200ms), NFR-003 (반응형: 모바일 2열, 태블릿 3열, 데스크톱 4열)
  • 기술 노트:
    • 기술 선택: Next.js SSR + 클라이언트 필터링 하이브리드
    • 대안: CSR 전체 (초기 로딩 느림), SSG (카테고리 변경 시 재빌드 필요)
    • 선택 사유: 초기 로딩 성능(LCP)과 동적 필터링 양립
    • 제약: 패키지 1,000개 초과 시 서버 사이드 필터링으로 전환 필요

FN-002: 패키지 상세 조회

  • FR 근거: FR-002 (Must)
  • 기획 근거: 설치 전환의 결정적 페이지. 설치 명령어 복사, README 렌더링, 버전 히스토리를 한 화면에 제공하여 전환율 30% 달성을 지원한다.
  • IA 근거: 생략
  • 복잡도: 높음

화면 구성

영역 구성요소 설명
상단 패키지 헤더 아이콘, 이름, 한줄 설명, 퍼블리셔 정보, 카테고리 태그
상단 우측 설치 버튼 영역 CLI 복사 버튼 + 원클릭 설치 버튼
중앙 탭1 README GitHub README.md 렌더링 (마크다운)
중앙 탭2 버전 히스토리 전체 버전 목록 + CHANGELOG
중앙 탭3 리뷰 리뷰 목록 (FN-015 임베드)
사이드바 메타 정보 설치 수, 별점, 최신 버전, 라이선스, 리포 링크

입력

항목 타입 필수 검증 규칙
패키지 ID/slug string 예 URL 파라미터. 존재하지 않으면 404
탭 선택 enum 아니오 기본: README

처리

  • 정상 흐름:
    1. URL의 패키지 slug로 상세 정보 조회
    2. README.md를 마크다운→HTML 렌더링
    3. 메타 정보(설치 수, 별점 등) 사이드바에 표시
    4. 탭 전환 시 해당 콘텐츠 로드 (lazy loading)
  • 예외 처리:
    • 패키지 미공개(심사 중): 퍼블리셔 본인에게만 미리보기 표시, 타인에게 404
    • README 미존재: "아직 문서가 작성되지 않았습니다." 안내
  • 에러 처리:
    • 패키지 미존재: 404 에러 페이지 + 목록으로 돌아가기 링크
    • GitHub API 동기화 실패: 캐시된 이전 README 표시 + "최신 정보가 아닐 수 있습니다" 배지

출력

항목 타입 설명
패키지 상세 정보 object 이름, 설명, README HTML, 스크린샷
설치 명령어 string 복사 가능한 코드블록
버전 목록 array 버전번호, 날짜, 변경 로그
리뷰 요약 object 평균 별점, 총 리뷰 수

    • 제약: README 렌더링에 remark/rehype 파이프라인 필요. XSS 방지 sanitize 필수

FN-003: 패키지 검색

  • FR 근거: FR-003 (Must)
  • 기획 근거: 패키지 수 증가 시 검색이 탐색의 1순위 경로가 된다. 디바운스 + 자동완성으로 검색 경험을 최적화하고 전환율 30% 목표를 지원한다.
  • IA 근거: 생략
  • 복잡도: 높음

화면 구성

영역 구성요소 설명
GNB 내장 검색 입력 필드 아이콘 + placeholder "패키지 검색..."
드롭다운 자동완성 결과 최대 5건 패키지명 + 카테고리
드롭다운 하단 최근 검색어 최대 5건 (로컬스토리지)
검색 결과 페이지 결과 목록 FN-001 카드 그리드와 동일 레이아웃

입력

항목 타입 필수 검증 규칙
검색어 text 예 최소 2자, 최대 100자. 300ms 디바운스

처리

  • 정상 흐름:
    1. 사용자 입력 시 300ms 디바운스 후 자동완성 API 호출
    2. 패키지명, 설명, 태그에서 부분일치 검색
    3. 검색 결과를 관련도순으로 정렬
    4. 검색어를 로컬스토리지에 최근 검색어로 저장 (최대 5건, FIFO)
    5. Enter 또는 검색 버튼 클릭 시 결과 페이지로 이동
  • 예외 처리:
    • 결과 0건: "'{검색어}'에 대한 결과가 없습니다." + 추천 패키지 3건 표시
    • 검색어 2자 미만: 자동완성 미호출, 최근 검색어만 표시
  • 에러 처리:
    • API 실패: "검색 서비스에 일시적 문제가 있습니다." + 재시도 안내
    • 타임아웃: 이전 검색 결과 유지

출력

항목 타입 설명
자동완성 목록 array 패키지명 + 카테고리 (최대 5건)
검색 결과 array FN-001과 동일 카드 형태
최근 검색어 array 최근 5건 (로컬스토리지)

FN-020: 사용자 관리 (관리자)

  • FR 근거: FR-020 (Must)
  • 기획 근거: 역할 기반 접근 제어(일반/퍼블리셔/관리자)와 악성 사용자 밴 처리로 플랫폼 품질을 유지한다.
  • IA 근거: 생략
  • 복잡도: 중간

처리

  • 정상 흐름:
    1. 사용자 목록 조회 (검색: 닉네임/이메일, 필터: 역할/상태)
    2. 사용자 상세: 프로필 정보, 설치 이력, 등록 패키지, 리뷰 이력
    3. 역할 변경: 일반 ↔ 퍼블리셔 ↔ 관리자 (드롭다운)
    4. 비활성화(밴): 사유 입력 필수 → 해당 사용자 로그인 차단
  • 예외 처리:
    • 자기 자신 밴 시도: "자신을 비활성화할 수 없습니다."
    • 마지막 관리자 역할 변경: "최소 1명의 관리자가 필요합니다."
  • 에러 처리:
    • 역할 변경 실패: "역할 변경에 실패했습니다." + 재시도
