1. 문서 개요

항목 내용
프로젝트명 엘루오 허브 (Eluo Hub)
업종 SaaS / AI 자동화 도구 플랫폼
목적 AI 기획 자동화 도구(스킬/에이전트)를 검색, 설치, 관리할 수 있는 마켓플레이스 구축
범위 마켓플레이스 코어 + 사용자 관리 + 패키지 관리 + 리뷰 시스템 + 관리자 기능

4. 기능 요구사항 (FR)

마켓플레이스 코어

FR-### 요구사항명 설명 우선순위 AC (인수조건) QST 근거
FR-001 패키지 목록 조회 등록된 패키지를 카테고리/태그별로 목록 조회한다 Must 1. 패키지 목록이 카드 형태로 표시된다 2. 카테고리(파이프라인/기능/업종) 필터링이 작동한다 3. 정렬(인기순/최신순/평점순)이 작동한다 4. 페이지당 12개씩 무한스크롤 또는 페이지네이션으로 로드된다 Q-008
FR-002 패키지 상세 조회 패키지의 상세 정보(설명, 설치 방법, 버전, 리뷰)를 확인한다 Must 1. 패키지명, 설명, 스크린샷이 표시된다 2. 설치 명령어가 복사 가능한 코드블록으로 표시된다 3. 버전 히스토리(전체)가 표시된다 4. README.md 내용이 렌더링된다 Q-009, Q-012
FR-003 패키지 검색 키워드 기반으로 패키지를 검색한다 Must 1. 검색어 입력 시 300ms 디바운스 후 결과가 표시된다 2. 패키지명, 설명, 태그에서 검색된다 3. 검색 결과 0건 시 관련 추천이 표시된다 4. 최근 검색어가 5건까지 표시된다 Q-008
FR-004 패키지 설치 (CLI) CLI 명령어를 통해 패키지를 로컬에 설치한다 Must 1. "설치" 버튼 클릭 시 CLI 명령어가 클립보드에 복사된다 2. 설치 가이드(3단계 이내)가 표시된다 3. 의존 패키지가 있으면 함께 표시된다 4. 설치 성공/실패 상태가 기록된다 Q-009
FR-005 패키지 원클릭 설치 웹에서 원클릭으로 패키지 설치를 트리거한다 Should 1. "원클릭 설치" 버튼이 CLI 설치와 별도로 존재한다 2. Claude Code CLI가 로컬에 설치된 경우 자동 실행된다 3. CLI 미설치 시 설치 안내로 폴백된다 4. 설치 진행률이 실시간 표시된다 Q-009
FR-006 카테고리 관리 패키지를 다중 카테고리(파이프라인/기능/업종)로 분류한다 Must 1. 3가지 분류 축(파이프라인 단계, 기능 유형, 업종)이 존재한다 2. 패키지 1개에 복수 카테고리 태깅이 가능하다 3. 카테고리별 패키지 수가 표시된다 4. 빈 카테고리는 목록에서 숨겨진다 Q-008

FR-009 설치 이력 관리 사용자가 설치한 패키지 이력을 확인한다 Should 1. 마이페이지에서 설치한 패키지 목록이 시간순으로 표시된다 2. 패키지별 설치 버전과 최신 버전이 비교 표시된다 3. 업데이트 가능 패키지에 배지가 표시된다 Q-012

패키지 퍼블리싱

FR-### 요구사항명 설명 우선순위 AC (인수조건) QST 근거
FR-010 패키지 등록 새 패키지를 등록하고 심사를 요청한다 Must 1. GitHub 리포지토리 URL 입력으로 패키지 정보가 자동 수집된다 2. 패키지명, 설명, 카테고리, 태그를 입력/수정할 수 있다 3. README.md가 미리보기로 렌더링된다 4. 제출 시 "심사 대기" 상태로 등록된다 Q-010
FR-011 패키지 버전 관리 패키지의 버전을 등록/관리한다 Must 1. Git 태그 기반으로 버전이 자동 감지된다 2. 전체 버전 히스토리가 표시된다 3. 각 버전의 변경 로그(CHANGELOG)가 표시된다 4. 사용자는 특정 버전을 선택하여 설치할 수 있다 Q-012
