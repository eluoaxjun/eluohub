설치 가이드

3단계만 따라하면 팀 전체가 Eluo Hub 플러그인을 사용할 수 있습니다.

사전 요구사항

  • Claude Code v1.0.33 이상
  • /plugin 커맨드 지원 버전
  • 인터넷 연결 (GitHub 접근)
1

마켓플레이스 등록

Eluo Hub 마켓플레이스를 Claude Code에 등록합니다. 최초 1회만 실행하면 됩니다.

Claude Code 
/plugin marketplace add eluoaxjun/eluohub

실행 후 eluo-hub 마켓플레이스가 등록되었다는 메시지가 표시됩니다.

2

플러그인 설치

필요한 플러그인을 선택하여 설치합니다. Core는 모든 플러그인의 기반이므로 반드시 먼저 설치합니다.

필수 
/plugin install core@eluo-hub
선택 — 필요한 것만 설치 
/plugin install planning@eluo-hub
/plugin install design@eluo-hub
/plugin install publish@eluo-hub
/plugin install qa@eluo-hub
/plugin install ops@eluo-hub

추천 조합

용도플러그인 조합설명
구축 Core + Planning + Design + Publish + QA 신규 사이트 구축. 기획부터 QA까지 전체 파이프라인
운영 Core + Planning + Publish + Ops 기존 사이트 유지운영. 요청 접수 + 수정 + 배포
전체 6개 전체 구축과 운영을 모두 커버

어떤 조합이 좋을지 모르겠다면 설정 생성기에서 프리셋을 선택해 보세요.

3

프로젝트 설정 (선택)

팀 프로젝트라면 .claude/settings.json에 설정을 추가하여 팀원이 자동으로 플러그인을 안내받도록 합니다.

.claude/settings.json 
{
  "extraKnownMarketplaces": {
    "eluo-hub": {
      "source": { "source": "github", "repo": "eluoaxjun/eluohub" }
    }
  },
  "enabledPlugins": {
    "core@eluo-hub": true,
    "planning@eluo-hub": true,
    "design@eluo-hub": true,
    "publish@eluo-hub": true,
    "qa@eluo-hub": true,
    "ops@eluo-hub": true
  }
}

이 설정을 Git에 커밋하면, 팀원이 프로젝트를 열 때 자동으로 플러그인 설치 안내를 받습니다. 설정 생성기에서 원하는 플러그인을 선택하면 이 JSON을 자동으로 생성해 줍니다.

사용법

설치 후 Claude Code에서 슬래시 커맨드로 각 기능을 실행합니다.

기획 파이프라인 
/planning:plan       ← 기획 전체 (QST → REQ → FN → IA → WBS)
/planning:qst        ← 고객질의서 단독 실행
/planning:req        ← 요구사항정의서 단독 실행
디자인 → 퍼블리싱 → QA 
/design:design       ← 디자인 전체 (벤치마킹 → 시안)
/publish:publish     ← 퍼블리싱 (Markup → Style → Interaction)
/qa:qa-run           ← QA 전체 (기능 + 접근성 + 성능)
유지운영 
/ops:maintenance     ← 운영 요청 접수 + 티켓 생성 + Notion 동기화

전체 커맨드 목록은 커맨드 레퍼런스를 참조하세요.

자동 라우팅

Core 플러그인에 포함된 PM Router가 자연어 요청을 분석하여 적절한 파이프라인으로 자동 분기합니다. 슬래시 커맨드를 외우지 않아도 됩니다.

자연어 예시 
"이 사이트 기획해줘"        → Planning 오케스트레이터
"홈페이지 디자인 해주세요"     → Design 오케스트레이터
"배너 이미지 교체해주세요"     → Ops 오케스트레이터 (운영)
"QA 돌려줘"                  → QA 오케스트레이터

FAQ

Q. 플러그인을 제거하려면?

/plugin uninstall planning@eluo-hub

Core를 제거하면 다른 플러그인이 정상 작동하지 않습니다. Core는 항상 유지하세요.

Q. 업데이트는 어떻게?

동일한 /plugin install 커맨드를 다시 실행하면 최신 버전으로 업데이트됩니다.

Q. 팀원마다 다른 플러그인을 쓸 수 있나요?

네. .claude/settings.json은 프로젝트 단위 설정이지만, 개인 환경에서 추가 플러그인을 설치해도 충돌하지 않습니다.

Q. 오프라인에서도 사용 가능한가요?

최초 설치 시 GitHub 접근이 필요하지만, 설치된 플러그인은 로컬에 캐싱되어 이후에는 오프라인에서도 동작합니다.
