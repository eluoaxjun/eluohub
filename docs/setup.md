# 최초 세팅 가이드

> 엘루오 허브 스킬을 다운로드하고 바로 사용하기까지의 과정을 안내합니다.

---

## 사전 요구사항

| 항목 | 최소 버전 | 확인 방법 |
|------|----------|----------|
| Claude Code | v2.1 이상 | `claude --version` |
| Node.js | 18 이상 | `node --version` |

---

## 1단계: 스킬 폴더 다운로드

필요한 스킬 폴더를 다운로드합니다. 각 폴더는 **자기완결형**이므로 하나만 받아도 바로 사용할 수 있습니다.

**예시**

| 스킬 | 폴더명 | 설명 |
|------|--------|------|
| 고객질의서 | `plan-qst/` | 고객 요구사항 수집 |
| 요구사항정의서 | `plan-req/` | 기능·비기능 요구사항 정의 |
| 기능정의서 | `plan-fn/` | 입력-처리-출력 상세 명세 |

> 전체 스킬 목록은 [스킬 카탈로그](index.html)를 참고하세요.

---

## 2단계: 스킬 폴더에서 Claude Code 실행

다운로드한 스킬 폴더를 열고 Claude Code를 실행합니다.

**예시**

```
plan-req/                    ← 이 폴더에서 Claude Code 실행
├── .claude/                 ← 스킬 설정 (자동 인식, 수정 불필요)
├── README.md                ← 사용 설명
├── input/                   ← 참고 자료 (제안서, 시안, RFP 등)
└── output/                  ← 산출물이 여기에 생성됩니다
```

### 참고 자료 넣기

`input/` 폴더에 아래와 같은 자료를 넣으면 산출물 품질이 향상됩니다:

- 제안서, RFP
- 디자인 시안 (이미지, PDF)
- 현행 사이트 분석 자료
- 벤치마킹 레퍼런스

### PROJECT.md (자동 생성)

`PROJECT.md`는 **직접 작성할 필요 없습니다.** 스킬을 처음 실행하면 프로젝트 유형·고객사·요구사항 등을 질문한 뒤 자동으로 생성됩니다.

> 원하면 직접 작성해도 됩니다:
>
> ```markdown
> # 프로젝트명
>
> ## 개요
> - 고객사: OOO
> - 프로젝트 유형: 신규 구축 / 리뉴얼 / 운영
> - 오픈 예정일: YYYY-MM-DD
>
> ## 주요 요구사항
> - (고객이 전달한 핵심 내용을 기술)
> ```

---

## 3단계: 동작 테스트

스킬 폴더에서 Claude Code를 실행한 뒤, 자연어로 요청합니다.

**예시**

```
"요구사항 정리해줘"
```

`output/` 폴더에 산출물 파일이 생성되면 설치가 완료된 것입니다.

---

## MCP 서버

대부분의 스킬은 **MCP 서버가 필요 없습니다.**

일부 스킬(디자인 벤치마킹, QA 성능 테스트 등)은 MCP 서버(Playwright 등)가 필요할 수 있습니다. 해당 스킬의 README.md에 설치 안내가 포함됩니다.

---

## 업데이트

새 버전을 배포받으면, 스킬 폴더의 `.claude/` 내용물을 덮어쓰기합니다. `input/`, `output/` 등 작업 데이터는 영향 없습니다.

---

## 제거

스킬 폴더를 삭제합니다. 산출물을 보관하려면 `output/` 폴더만 별도로 백업하세요.

---

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 자연어 요청에 반응 안 함 | Claude Code 재시작 필요 | Claude Code 완전 종료(`Ctrl+C`) 후 재시작 |
| 산출물이 생성되지 않음 | 스킬 폴더 밖에서 실행 | 스킬 폴더에서 Claude Code 실행 |
| 스킬이 동작 안 함 | `.claude/` 경로 오류 | `.claude/skills/` 하위에 스킬 폴더가 있는지 확인 |
