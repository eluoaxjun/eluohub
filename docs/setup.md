# 최초 세팅 가이드

> 엘루오 허브를 처음 설치하고 프로젝트를 시작하기까지의 전체 과정을 안내합니다.

---

## 사전 요구사항

| 항목 | 최소 버전 | 확인 방법 |
|------|----------|----------|
| Claude Code | v2.1 이상 | `claude --version` |
| Node.js | 18 이상 | `node --version` |

> Claude Code를 설치한 뒤 **1회 이상 실행**해야 `~/.claude/` 디렉토리가 생성됩니다.

---

## 1단계: 파일 복사

배포 폴더는 **공통**과 **스킬**로 나뉘어 있습니다.

### 공통 (필수)

모든 스킬이 공유하는 기반 파일입니다. 반드시 먼저 복사합니다.

```
배포 폴더/공통/
├── agents/          →  ~/.claude/agents/
├── rules/           →  ~/.claude/rules/
├── hooks/           →  ~/.claude/hooks/
├── scripts/         →  ~/.claude/scripts/
├── standards.md     →  ~/.claude/standards.md
└── handoff-protocol.md → ~/.claude/handoff-protocol.md
```

| 폴더 | 역할 |
|------|------|
| `agents/` | Planning Orchestrator, Reviewer (파이프라인 실행·검수) |
| `rules/` | 품질 기준, ID 추적성, PM 프로토콜 |
| `hooks/` + `scripts/` | 키워드 감지, 세션 초기화 자동 실행 |

### 스킬 (선택)

필요한 스킬만 골라서 `~/.claude/skills/`에 복사합니다.

| 스킬 | 폴더명 | 설명 |
|------|--------|------|
| 전체 파이프라인 | `plan/` | QST→REQ→FN→IA→WBS→SB 순서대로 실행 |
| 고객질의서 | `plan-qst/` | 고객 요구사항 수집 |
| 요구사항정의서 | `plan-req/` | 기능·비기능 요구사항 정의 |
| 기능정의서 | `plan-fn/` | 입력-처리-출력 상세 명세 |
| 정보구조설계 | `plan-ia/` | 페이지 구조·메뉴·URL 설계 |
| 작업분해구조 | `plan-wbs/` | 일정·담당자·크리티컬 패스 |
| 화면설계서 | `plan-SB/` | 레이아웃·구성 요소 정의 |
| PM 라우터 | `pm-router/` | 자연어 요청을 스킬로 자동 분기 |

**예시**: 요구사항정의서와 기능정의서만 사용하려면

```
~/.claude/skills/plan-req/
~/.claude/skills/plan-fn/
```

> PM 라우터(`pm-router/`)를 함께 복사하면 "요구사항 정리해줘"처럼 자연어로 실행됩니다.

### 복사 시 주의사항

- `~/.claude/`에 같은 이름의 폴더가 이미 있으면, 기존 파일은 유지하고 새 파일만 추가합니다
- 기존에 직접 만든 파일과는 파일명이 다르므로 충돌하지 않습니다

---

## 2단계: Claude Code 재시작

파일 복사 후 Claude Code를 **완전히 종료하고 다시 실행**합니다.

> 정상적으로 인식되지 않으면 Claude Code를 완전히 종료(`Ctrl+C` 또는 `/exit`) 후 재시작하세요.

---

## 3단계: 프로젝트 폴더 준비

프로젝트 폴더에 아래 구조를 만듭니다.

```
내 프로젝트/
├── PROJECT.md          ← 프로젝트 기본 정보
├── input/              ← 참고 자료 (시안, 제안서, 레퍼런스 등)
└── output/
    └── planning/       ← 산출물이 여기에 생성됩니다
```

### PROJECT.md 작성

프로젝트의 기본 정보를 작성합니다. 이 파일이 있으면 산출물 생성 시 자동으로 참조됩니다.

```markdown
# 프로젝트명

## 개요
- 고객사: OOO
- 프로젝트 유형: 신규 구축 / 리뉴얼 / 운영
- 오픈 예정일: YYYY-MM-DD

## 주요 요구사항
- (고객이 전달한 핵심 내용을 기술)

## 참고 자료
- input/ 폴더 내 파일 목록
```

### input/ 폴더 활용

아래와 같은 자료를 넣으면 산출물 품질이 향상됩니다:

- 제안서, RFP
- 디자인 시안 (이미지, PDF)
- 현행 사이트 분석 자료
- 벤치마킹 레퍼런스

---

## 4단계: 동작 테스트

프로젝트 폴더에서 Claude Code를 실행한 뒤, 간단한 요청으로 정상 동작을 확인합니다.

```
"요구사항 정리해줘"
```

`output/planning/` 폴더에 `REQ_*.md` 파일이 생성되면 설치가 완료된 것입니다.

---

## 개인 설정은 필요한가요?

**필요 없습니다.** 공통 폴더를 복사하면 품질 기준·에이전트·훅이 모두 포함됩니다.

> `~/.claude/CLAUDE.md`나 `~/.claude/settings.json`을 별도로 수정할 필요가 없습니다.

---

## 업데이트

새 버전을 배포받으면, 동일하게 `~/.claude/`에 덮어쓰기합니다.

---

## 제거

복사했던 파일을 삭제합니다. 스킬을 개별로 복사했다면 해당 파일만 삭제하면 됩니다.

---

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 자연어 요청에 반응 안 함 | 재시작 누락 | Claude Code 완전 종료 후 재시작 |
| 산출물이 생성되지 않음 | 프로젝트 폴더 밖에서 실행 | 프로젝트 폴더에서 Claude Code 실행 |
| `~/.claude` 디렉토리 없음 | Claude Code 미설치 | Claude Code 설치 후 1회 실행 |
| 스킬이 동작 안 함 | 파일 경로 오류 | `~/.claude/skills/` 하위에 폴더가 정확히 복사되었는지 확인 |
