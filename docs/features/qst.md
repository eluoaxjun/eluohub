# 고객질의서 (QST)

> 프로젝트를 시작하기 전에, 고객에게 물어봐야 할 질문들을 자동으로 정리해줍니다.

---

## 한 줄 요약

프로젝트 착수 전, 고객의 비즈니스 목표·타겟·기능·일정을 **구조화된 질의**로 수집하는 문서입니다.

---

## 이런 상황에서 써요

- 프로젝트 킥오프 전 고객 요구사항을 체계적으로 수집해야 할 때
- 구두로만 전달받은 요구사항을 문서화해야 할 때
- 이해관계자 간 인식 차이를 사전에 줄여야 할 때

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"고객한테 뭘 물어봐야 하지?"
"질의서 만들어줘"
"확인할 것 정리해줘"
```

---

## 어떤 결과가 나오나요?

**파일**: `output/planning/QST_{프로젝트코드}_v1.md`

질의서에 포함되는 내용:

| 항목 | 수집 내용 |
|------|----------|
| 비즈니스 목표 | KPI, 전환 목표, 성과 지표 |
| 타겟 사용자 | 주요 사용자군, 디바이스 비율, 사용 맥락 |
| 핵심 기능 | Must/Should/Could 기능 분류 |
| 콘텐츠 | 콘텐츠 유형, 소스, 갱신 주기 |
| 일정/예산 | 마일스톤, 예산 범위, 리소스 제약 |
| 기술 환경 | 호스팅, CMS, 외부 연동, 인증 방식 |
| 디자인 방향 | 레퍼런스 사이트, 브랜드 가이드, 톤앤매너 |

각 질문은 **선택형(A/B/C)**으로 되어 있어서 고객이 쉽게 답할 수 있습니다.

---

## 산출물 예시

```markdown
## Q-001. 프로젝트의 주요 목표는 무엇입니까?

> 카테고리: 비즈니스 목표

- [ ] A. 신규 고객 유입 증가
- [ ] B. 기존 고객 서비스 개선
- [ ] C. 브랜드 인지도 향상
- [ ] D. 직접 입력: _______________

💡 참고: 목표가 여러 개면 우선순위를 정해주세요.
```
