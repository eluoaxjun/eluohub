# 작업분해구조 (WBS)

> 프로젝트 일정과 작업을 분해하고, 누가 언제까지 할지 배정해줍니다.

---

## 한 줄 요약

프로젝트를 단위 작업으로 분해하고, **일정·담당자·의존관계·크리티컬 패스**를 도출하는 문서입니다.

---

## 이런 상황에서 써요

- 프로젝트 전체 공수를 산정하고 일정을 수립해야 할 때
- 직군별(기획/디자인/퍼블리싱/개발/QA) 작업을 배분해야 할 때
- 고객에게 마일스톤 기반 일정표를 제출해야 할 때
- 병목 구간과 크리티컬 패스를 식별해야 할 때

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"일정 짜줘"
"얼마나 걸리는지 산정해줘"
"작업 분해해줘"
```

---

## 어떤 결과가 나오나요?

**파일**: `output/planning/WBS_{프로젝트코드}_v1.md`

문서에 포함되는 내용:

| 섹션 | 설명 |
|------|------|
| 작업 목록 | 단계별 작업 분해 (WBS ID 체계) |
| 일정 산정 | 작업별 소요 기간과 선행 의존관계 |
| 담당자 배정 | 직군별 리소스 배분 |
| 간트 차트 | 주 단위 일정 시각화 |
| 크리티컬 패스 | 전체 일정을 좌우하는 최장 경로 |
| 마일스톤 | 단계별 점검 시점과 산출물 기준 |

---

## 산출물 예시

### 작업 목록

```markdown
| WBS ID | 작업명 | 기간 | 담당 | 선행작업 |
|--------|--------|------|------|---------|
| 1.1 | 기획 | 2주 | PM | — |
| 1.2 | 디자인 | 3주 | 디자이너 | 1.1 |
| 1.3 | 퍼블리싱 | 2주 | 퍼블리셔 | 1.2 |
| 1.4 | 개발 | 4주 | 개발자 | 1.3 |
| 1.5 | QA/테스트 | 1주 | QA | 1.4 |
```

### 간트 차트 (텍스트)

```
         W1   W2   W3   W4   W5   W6   W7   W8   W9  W10  W11  W12
기획     ████████
디자인          ████████████
퍼블리싱                   ████████
개발                              ████████████████
QA                                                 ████
                                                        ↑ 오픈
```

### 마일스톤

```markdown
| # | 마일스톤 | 일자 | 산출물 |
|---|---------|------|--------|
| M1 | 기획 완료 | W2 금 | REQ, FN, IA 확정 |
| M2 | 디자인 확정 | W5 금 | 시안 최종 승인 |
| M3 | 개발 완료 | W10 금 | 전 페이지 개발 완료 |
| M4 | 오픈 | W12 월 | 사이트 오픈 |
```
