# 화면설계서 (SB)

> JSON 데이터를 입력받아 표준 양식의 화면설계서를 HTML/PDF로 자동 생성합니다.

---

## 한 줄 요약

각 화면의 **레이아웃·구성 요소·동작 명세**를 시각적으로 정의하는 문서입니다. 디자인 착수의 기준이 됩니다.

---

## 이런 상황에서 써요

- 디자이너에게 전달할 화면 단위 구성 명세가 필요할 때
- 기능정의서(FN)의 내용을 화면 레이아웃으로 구체화해야 할 때
- 표준 양식의 화면설계서를 산출물로 제출해야 할 때
- 기존 양식(테마)을 활용하거나, 기본(default) 양식으로 빠르게 문서를 만들고 싶을 때

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"화면설계서 만들어줘"
"화면 구성 정리해줘"
```

JSON 데이터 파일이 `data/` 또는 `input/`에 있으면 자동 감지하여 바로 생성합니다.
파일이 없으면 대화형으로 정보를 수집한 뒤 생성합니다.

---

## 주요 기능

### HTML → PDF 변환

화면설계서는 **HTML로 먼저 생성**된 뒤, 자동으로 **PDF로 변환**됩니다.

- HTML: 브라우저에서 바로 확인 가능한 인터랙티브 문서
- PDF: 인쇄·공유용 고정 레이아웃 문서 (Playwright 기반 변환)
- Playwright 미설치 환경에서는 HTML만 생성 (graceful fallback)

### 테마 프리셋 (양식 선택)

프로젝트에 맞는 양식을 선택하여 사용할 수 있습니다.

| 양식 | 설명 |
|------|------|
| `default` | 기본 양식. 별도 설정 없이 바로 사용 가능 |
| `kmvno` | 테스트용 샘플 양식 |
| 커스텀 | `themes/` 폴더에 JSON 파일 추가로 자체 양식 생성 가능 |

별도 지정이 없으면 `default` 양식이 자동 적용됩니다.
주 색상(`primaryColor`), 강조 색상(`accentColor`), 로고도 개별 오버라이드할 수 있습니다.

### 4종 프레임 타입

하나의 문서 안에서 용도별로 다른 프레임 유형을 사용합니다.

| 프레임 타입 | 용도 |
|------------|------|
| Design | 와이어프레임 + 수정 영역 설명 (기본) |
| Description | 설명 중심 (와이어프레임 없음) |
| MSG Case | 메시지/다이얼로그 케이스 정의 |
| Component Guide | UI 컴포넌트 상태·사양 가이드 |

### v1/v2 스키마 자동 정규화

기존 v1 스키마(KMVNO 전용)로 작성된 JSON도 v2(범용)로 자동 변환됩니다.
기존 데이터를 수정할 필요 없이 그대로 사용할 수 있습니다.

---

## 어떤 결과가 나오나요?

**파일**: `output/{프로젝트명}/{프로젝트명}_SB_{날짜}_v1.0.html` + `.pdf`

문서에 포함되는 프레임:

| 프레임 | 설명 |
|--------|------|
| Cover | 프로젝트 표지 (로고, 과제명, 버전, 검토/승인) |
| History | 변경 이력 테이블 |
| Overview | 업무 개요 또는 요약 |
| Screen | 화면별 설계 (타입에 따라 Design/Description/MSG/Component) |
| Divider | 섹션 구분 페이지 |
| End of Document | 문서 종료 표시 |

---

## FN과 SB의 차이

| 문서 | 초점 |
|------|------|
| FN (기능정의서) | 기능이 **어떻게 동작**하는지 — 입력·처리·출력·검증 |
| SB (화면설계서) | 화면이 **어떻게 구성**되는지 — 레이아웃·요소 배치·인터랙션 |

FN은 **동작 명세**, SB는 **시각 명세**입니다.

## 팁 & 주의사항

### PDF 생성 환경 요건

PDF 변환은 내부적으로 `scripts/generate.js` (Node.js + Playwright)를 사용합니다.

- **Node.js가 설치되어 있어야** PDF 자동 생성이 됩니다
- 에이전트 환경에 따라 스크립트가 자동 실행되지 않을 수 있습니다. 이 경우 직접 실행하세요:

```bash
node ".claude/skills/plan-sb/scripts/generate.js" \
  output/{프로젝트명}/{파일명}.html
```

- PDF 생성이 실패해도 **HTML은 정상 생성**됩니다 (graceful fallback)
- HTML 파일을 Chrome에서 열고 인쇄(Ctrl+P) → PDF 저장으로 수동 변환할 수 있습니다

### 운영 모드 — URL 캡처 방식

운영 모드에서 기존 화면 URL을 제공하면 Playwright로 스크린샷을 캡처하여 화면설계서에 삽입합니다.

- Playwright가 설치되어 있어야 캡처가 작동합니다: `npx playwright install chromium`
- 캡처 실패 시 와이어프레임 폴백으로 자동 전환됩니다
- 캡처 이미지는 `input/` 폴더에 PNG로 저장됩니다

### 구축 모드 — 와이어프레임 방식

구축(신규 서비스) 모드에서는 URL 없이 대화형으로 정보를 수집하여 와이어프레임을 생성합니다.

- GNB, 카드 그리드, 검색 필터 등 포함할 화면 요소를 프롬프트에 명시하면 더 정확한 와이어프레임이 생성됩니다
- 마커 번호(①②③)와 Description 번호가 일치하는지 확인하세요
