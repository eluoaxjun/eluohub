# 화면설계서 (SB)

> 각 화면의 레이아웃과 구성 요소를 표준 양식 문서로 만들어줍니다.

---

## 한 줄 요약

각 화면의 **레이아웃·구성 요소·동작 명세**를 시각적으로 정의하는 문서입니다. 디자인 착수의 기준이 됩니다.

---

## 이런 상황에서 써요

- 디자이너에게 전달할 화면 단위 구성 명세가 필요할 때
- 기능정의서(FN)의 내용을 화면 레이아웃으로 구체화해야 할 때
- 표준 양식의 화면설계서를 산출물로 제출해야 할 때

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"화면설계서 만들어줘"
"화면 구성 정리해줘"
```

---

## 어떤 결과가 나오나요?

**파일**: `output/planning/SB_{프로젝트코드}_v1.html` (HTML/PDF)

문서에 포함되는 내용:

| 섹션 | 설명 |
|------|------|
| 표지 | 프로젝트명, 버전, 작성일 |
| 변경 이력 | 수정 내역 추적 |
| 화면 목록 | 전체 화면 인덱스 |
| 화면별 설계 | 레이아웃 + 구성 요소 + 동작 설명 |

## 산출물 예시

### 화면 한 건

```
┌─────────────────────────────────┐
│         [헤더 / GNB]            │
├─────────────────────────────────┤
│  장르 필터 (소설│에세이│만화)    │
├─────────────────────────────────┤
│ ┌───────┐ ┌───────┐ ┌───────┐  │
│ │ 카드1  │ │ 카드2  │ │ 카드3  │  │
│ │ 이미지 │ │ 이미지 │ │ 이미지 │  │
│ │ 제목   │ │ 제목   │ │ 제목   │  │
│ │ 가격   │ │ 가격   │ │ 가격   │  │
│ └───────┘ └───────┘ └───────┘  │
├─────────────────────────────────┤
│       [1] [2] [3] ... [>]       │
├─────────────────────────────────┤
│           [푸터]                 │
└─────────────────────────────────┘
```

**화면 정보**:
```markdown
| 항목 | 내용 |
|------|------|
| 화면 ID | SC-001 |
| 화면명 | 도서 목록 |
| 연관 FN | FN-001 |
| 레이아웃 | 3단 카드 그리드 |
```

---

## FN과 SB의 차이

| 문서 | 초점 |
|------|------|
| FN (기능정의서) | 기능이 **어떻게 동작**하는지 — 입력·처리·출력·검증 |
| SB (화면설계서) | 화면이 **어떻게 구성**되는지 — 레이아웃·요소 배치·인터랙션 |

FN은 **동작 명세**, SB는 **시각 명세**입니다.
