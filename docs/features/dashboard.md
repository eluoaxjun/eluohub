# 대시보드 (Dashboard)

> 지금까지 만든 모든 산출물의 현황을 한 페이지로 요약해줍니다.

---

## 한 줄 요약

**성적표**입니다. 프로젝트가 지금 어디까지 왔는지, 뭐가 잘 되고 있고 뭐가 문제인지 한눈에 보여줘요.

---

## 이런 상황에서 써요

- "프로젝트 지금 어디까지 왔어?" 라고 물어볼 때
- 상사나 고객에게 진행 현황을 보고할 때
- 이슈나 미확인 사항이 몇 개인지 파악하고 싶을 때
- 산출물이 다 만들어졌는지 점검할 때

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"전체 현황 정리해줘"
"지금까지 뭐 했는지 보여줘"
"프로젝트 현황 요약해줘"
```

---

## 어떤 결과가 나오나요?

**파일**: `output/planning/Dashboard_{프로젝트코드}_v1.md`

문서에 포함되는 내용:

| 섹션 | 설명 | 비유 |
|------|------|------|
| 산출물 현황 | 각 문서의 완성 상태 | 과목별 성적 |
| 수량 요약 | FR 몇 건, FN 몇 건 등 | 총점 |
| KPI | 주요 지표 달성 현황 | 목표 대비 달성률 |
| 미확인 항목 | 아직 결정 안 된 것들 | 숙제 남은 것 |
| 이슈 | 해결해야 할 문제들 | 오답 노트 |
| 마일스톤 | 중간 점검 진행률 | 시험 일정 |

---

## 산출물 예시

### 산출물 현황

```markdown
| 산출물 | 파일 | 상태 | 버전 |
|--------|------|------|------|
| QST (고객질의서) | QST_OBS_v1.md | 완료 | v1.0 |
| REQ (요구사항) | REQ_OBS_v1.md | 완료 | v1.1 |
| FN (기능정의서) | FN_OBS_v1.md | 완료 | v1.1 |
| IA (정보구조) | — | 미작성 | — |
| WBS (일정) | — | 미작성 | — |
```

### 수량 요약

```markdown
| 항목 | 건수 |
|------|------|
| 기능 요구사항 (FR) | 18건 |
| 비기능 요구사항 (NFR) | 8건 |
| 기능 정의 (FN) | 15건 |
| 미확인 항목 [TBD] | 9건 |
| 알려진 이슈 | 3건 |
```

### 미확인 항목

```markdown
| # | 항목 | 관련 문서 | 상태 |
|---|------|----------|------|
| 1 | 결제 PG사 선택 (토스 vs KG) | REQ FR-007 | 미결정 |
| 2 | 멤버십 등급 기준 | REQ FR-010 | 미확인 |
| 3 | 재고 연동 방식 | REQ 제약사항 | 미협의 |
```

---

## 이 문서를 읽는 사람은?

- **PM**: 전체 진행 상황 관리, 이슈 추적
- **고객/상사**: 프로젝트가 잘 되고 있는지 확인
- **팀원**: 내가 맡은 부분 외에 전체 그림 파악

---

## 참고

대시보드는 `output/planning/` 폴더에 있는 산출물을 자동으로 읽어서 만들기 때문에,
**산출물이 많을수록 대시보드의 정보가 풍부해집니다.**
