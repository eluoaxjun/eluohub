# 엘루오 허브 — 기능 가이드

> 웹사이트를 만들 때 필요한 기획 문서를 자동으로 만들어주는 도구입니다.

---

## 이게 뭔가요?

웹사이트를 만들려면 "뭘 만들지?", "어떤 기능이 필요하지?", "언제까지 만들지?" 같은 것들을 미리 정리해야 합니다.
이 작업을 **기획**이라고 하는데, 보통 사람이 직접 문서를 하나하나 작성합니다.

엘루오 허브는 이 기획 문서를 **AI가 자동으로 만들어주는 도구**입니다.

---

## 어떤 기능이 있나요?

| 순서 | 기능 | 설명 |
|:---:|------|------|
| 1 | [고객질의서 (QST)](qst.md) | 프로젝트 착수 전 고객 요구사항을 구조화된 질의로 수집 |
| 2 | [요구사항정의서 (REQ)](req.md) | 기능·비기능 요구사항과 인수조건을 체계적으로 정의 |
| 3 | [기능정의서 (FN)](fn.md) | 각 기능의 입력-처리-출력과 검증 기준을 상세 명세 |
| 4 | [정보구조설계 (IA)](ia.md) | 사이트의 페이지 계층, 메뉴 체계, URL 구조를 설계 |
| 5 | [작업분해구조 (WBS)](wbs.md) | 작업 분해, 일정 산정, 담당자 배정, 크리티컬 패스 도출 |
| 6 | [화면설계서 (SB)](sb.md) | 화면별 레이아웃과 구성 요소를 시각적으로 정의 |

---

## 사용 방법

채팅창에 자연스럽게 말하면 됩니다. 정해진 문구가 아니라 **의도를 파악**해서 동작합니다.

```
"기획 문서 만들어줘"       → 전체 파이프라인 순서대로 실행
"요구사항 정리해줘"        → REQ만 실행
"사이트맵 좀 잡아줘"       → IA만 실행
```

위 문장은 예시일 뿐입니다. 비슷한 맥락의 말이면 자동으로 인식합니다.

> 각 기능의 상세 설명 페이지에도 어떤 말에 반응하는지 참고 예시가 있습니다.

---

## 문서는 어디에 저장되나요?

모든 산출물은 프로젝트 폴더 안에 자동 저장됩니다.

```
내 프로젝트/
├── input/              ← 참고 자료를 넣는 곳
└── output/
    └── planning/       ← 여기에 문서가 만들어집니다
        ├── QST_*.md
        ├── REQ_*.md
        ├── FN_*.md
        ├── IA_*.md
        └── WBS_*.md
```

---

## 순서가 중요한가요?

네, 앞 단계의 문서가 다음 단계의 재료가 됩니다.

```
QST(질문) → REQ(목록) → FN(상세) → IA(구조) → WBS(일정) → SB(화면)
```

- QST 없이 REQ를 만들면? → 가능하지만, 빠진 내용이 생길 수 있습니다
- REQ 없이 FN을 만들면? → 가능하지만, 기능 누락 위험이 있습니다

**"기획 문서 만들어줘"로 전체 실행하면 순서를 신경 쓸 필요가 없습니다.**

---

## 구축과 운영, 두 가지 모드

엘루오 허브는 프로젝트 상황에 따라 **자동으로 모드를 판단**합니다.

| 모드 | 언제? | 설명 | 비유 |
|------|------|------|------|
| **구축 모드** | 새 사이트를 처음 만들 때 | 전체 파이프라인을 순서대로 진행하거나, 필요한 문서만 골라서 생성 | 집을 새로 짓는 것 |
| **운영 모드** | 이미 있는 사이트를 수정할 때 | 변경 범위를 파악하고 필요한 문서만 생성 | 집을 리모델링하는 것 |

모드를 직접 선택할 필요 없이, 요청 내용을 보고 자동으로 판단합니다.

```
"홈페이지 새로 만들어줘"          → 구축 모드
"기능 정의만 해줘"                → 구축 모드 (개별 실행)
"메인 배너 수정해줘"              → 운영 모드
"이벤트 페이지 추가해줘"          → 운영 모드
```

> 구축 모드라고 해서 반드시 전체 파이프라인을 따를 필요는 없습니다.
> "기획 문서 만들어줘"로 전체를 한번에 돌릴 수도 있고, 필요한 문서만 개별로 만들 수도 있습니다.
