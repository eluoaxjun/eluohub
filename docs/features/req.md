# 요구사항정의서 (REQ)

> 이 프로젝트에서 만들어야 할 것들의 전체 목록을 자동으로 정리해줍니다.

---

## 한 줄 요약

프로젝트에 필요한 기능(FR)·비기능(NFR) 요구사항과 인수조건(AC)을 **빠짐없이 정의**한 문서입니다.

---

## 이런 상황에서 써요

- 고객 요구사항을 기능/비기능으로 분류하고 우선순위를 정해야 할 때
- 디자인·개발팀에 전달할 명확한 스펙 문서가 필요할 때
- 요구사항 누락을 방지하고, 프로젝트 범위를 합의해야 할 때

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"요구사항 정리해줘"
"뭘 만들어야 하는지 정리해줘"
"스펙 문서 만들어줘"
```

---

## 어떤 결과가 나오나요?

**파일**: `output/planning/REQ_{프로젝트코드}_v1.md`

문서에 포함되는 내용:

| 섹션 | 설명 | 예시 |
|------|------|------|
| 프로젝트 개요 | 프로젝트 기본 정보 | 프로젝트명, 고객사, 유형, 호스팅 |
| 비즈니스 목표 | 달성하려는 수치 목표 | "검색 유입 40% 증가" |
| 기능 요구사항 (FR) | 만들어야 할 기능 목록 | 게시판, 검색, 로그인 등 |
| 비기능 요구사항 (NFR) | 성능·보안 등 품질 조건 | "로딩 3초 이내" |
| 제약사항 | 지켜야 할 한계 | 예산, 일정, 기술 제약 |
| 추적성 매트릭스 | 요구사항 간 연결 관계 | FR-001 → FN-001 → IA-P100 |

---

## 산출물 예시

### 기능 요구사항 한 건

```markdown
| ID | 요구사항명 | 설명 | 우선순위 |
|----|-----------|------|---------|
| FR-001 | 도서 검색/필터 | 장르별로 도서를 분류하여 목록 표시 | High |
```

**완료기준 (AC)**:
```markdown
① 4개 장르 필터 정상 동작
② 가격 필터 적용 시 해당 범위 도서만 노출
③ 페이지당 10건 표시 + 총 건수 표시
```

### 비기능 요구사항 한 건

```markdown
| ID | 항목 | 요구수준 | 측정기준 |
|----|------|---------|---------|
| NFR-001 | 페이지 로딩 속도 | 3초 이내 | Lighthouse 90+ |
```

---

## 이 문서를 읽는 사람은?

- **기획자**: 빠진 요구사항이 없는지 점검
- **디자이너**: 어떤 화면이 필요한지 파악
- **개발자**: 무엇을 구현해야 하는지 확인
- **고객**: 우리가 제대로 이해했는지 검토
