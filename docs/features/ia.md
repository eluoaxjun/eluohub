# 정보구조설계 (IA)

> 웹사이트의 페이지 구조, 메뉴 배치, URL을 설계해줍니다.

---

## 한 줄 요약

사이트의 **페이지 계층·메뉴 체계·URL 구조**를 설계하는 문서입니다. 사이트맵과 네비게이션의 근거가 됩니다.

---

## 이런 상황에서 써요

- 사이트 전체 페이지 구조와 Depth를 설계해야 할 때
- GNB/LNB 메뉴 배치와 사용자 동선을 정의해야 할 때
- URL 체계(슬러그 규칙)를 표준화해야 할 때
- 신규 구축 또는 사이트 구조 리뉴얼 시

---

## 사용법

채팅창에 자연스럽게 말하면 됩니다:

```
"사이트맵 만들어줘"
"페이지 구조 잡아줘"
"메뉴 구조 설계해줘"
```

---

## 어떤 결과가 나오나요?

**파일**: `output/planning/IA_{프로젝트코드}_v1.md`

문서에 포함되는 내용:

| 섹션 | 설명 |
|------|------|
| 사이트맵 | 전체 페이지 트리 구조 (Depth별 계층) |
| 페이지 인벤토리 | 페이지별 ID, 유형, 연관 FR, URL 매핑 |
| 네비게이션 설계 | GNB/LNB/Footer 메뉴 구조와 사용자 동선 |
| 콘텐츠 매핑 | 각 페이지에 배치할 콘텐츠 유형과 소스 |
| URL 설계 | 슬러그 규칙, 파라미터 구조, 리다이렉트 정책 |

---

## 산출물 예시

### 사이트맵

```
홈 (IA-P001)
├── 도서 (IA-P100)
│   ├── 검색/목록 (IA-P101)
│   └── 상세 (IA-P110)
├── 추천 (IA-P200)
│   ├── 이달의 신간 (IA-P201)
│   ├── 베스트셀러 (IA-P202)
│   └── 테마별 추천 (IA-P203)
├── 이벤트 (IA-P300)
│   ├── 목록 (IA-P301)
│   └── 상세 (IA-P310)
└── 공통
    ├── 검색 (IA-P010)
    ├── FAQ (IA-P406)
    └── 이용약관 (IA-P020~P031)
```

### 페이지 인벤토리 (한 건)

```markdown
| IA ID | 페이지명 | Depth | 유형 | 연관 FR | URL |
|-------|---------|-------|------|---------|-----|
| IA-P100 | 도서 메인 | 2 | 목록형 | FR-001 | /books |
| IA-P110 | 도서 상세 | 3 | 상세형 | FR-002 | /books/{slug} |
```
