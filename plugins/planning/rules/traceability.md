# 산출물 간 ID 추적성 기준

모든 프로젝트에서 산출물 간 ID 추적 체인을 유지해야 한다.

## 추적 체인
```
FR-### → FN-### → UI-### → TC-###
```
- 모든 FN은 최소 1개 FR을 참조해야 하며, 모든 UI는 최소 1개 FN을 참조해야 한다
- orphan(미연결) ID 0건이 기본 목표

## NFR 추적
- 각 NFR-###에 대해 QA에서 최소 1건 이상의 TC 매핑 필수
- FN 상세 정의 시 관련 NFR-### ID를 명시적으로 참조

## QA TC ID 채번 규칙
- 기능 테스트: TC-F-### (예: TC-F-001)
- 접근성 테스트: TC-A-### (예: TC-A-001)
- 성능 테스트: TC-P-### (예: TC-P-001)
- 회귀 테스트: TC-R-### (예: TC-R-001)
- 비주얼 리그레션: TC-V-### (예: TC-V-001) — 스크린샷 베이스라인 대비 시각 변경 검증
- 모든 TC에 FR/FN/UI 참조 필수 (TC-V는 UI-### 참조 필수, FR/FN은 선택)

## 퍼블리싱 추적성
- HTML 각 섹션 시작 주석에 FR/FN/UI ID 명시 (예: `<!-- 섹션: 히어로 (FR-006 / FN-006 / UI-006) -->`)
- JS 파일 상단 주석에 담당 FN-### 전수 기재
- CSS 섹션 주석에 UI-### 참조

## FN 검증 기준 전파
- 모든 FN에 해당 FR의 인수조건(AC)을 검증기준으로 포함 (복잡도 무관 필수)
- REQ AC가 FN 검증기준으로 내려오지 않으면 QA에서 Pass/Fail 판단 근거가 부재하므로 반드시 전파

## 개별 실행 조항 (런타임 적응형)

스킬이 선행 산출물 없이 개별 실행될 때의 추적성 규칙이다.

### 원칙
- **선행 파일 미존재 시 역방향 추적을 생략한다** — 없는 ID를 추측하지 않는다
- **자체 ID 체계는 유지한다** — FN-###, IA-P### 등 산출물 내부 ID는 정상 부여
- **순방향 추적은 자동 활성화된다** — 독립 모드로 생성한 FN을 후속 IA가 읽으면 FN→IA 추적이 자동으로 적용

### 모드별 적용 범위

| 추적 유형 | 연계 모드 | 독립 모드 |
|-----------|----------|----------|
| FR-### → FN-### | 필수 | N/A (REQ 미존재) |
| FN-### → UI-### | 필수 | 필수 (FN 자체 ID 존재) |
| Q-### → FR-### | 필수 | N/A (QST 미존재) |
| TC → FR/FN/UI 참조 | 필수 | 존재하는 ID만 참조 |
| 추적성 매트릭스 | 필수 | 생략 |
| 퍼블리싱 주석 ID | FR/FN/UI 전수 | 존재하는 ID만 |

### 체크리스트 처리
- `[연계 모드 전용]` 표기된 체크 항목은 독립 모드에서 N/A 처리
- 독립 모드 합격 기준: `[연계 모드 전용]` 제외 항목 전수 Pass
