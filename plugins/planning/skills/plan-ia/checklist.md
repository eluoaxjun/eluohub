# IA 품질 체크리스트

## § 1. 입력 분석
- [ ] 프로젝트명 확인
- [ ] 프로젝트 유형 (구축/운영) 판별
- [ ] 업종 카테고리 분류
- [ ] 타겟 사용자 식별
- [ ] REQ/FN 산출물의 ID 전수 추출 (해당 시)
- [ ] 기능별 우선순위 분류 완료
- [ ] 최소 10개 이상 페이지 후보 도출
- [ ] 각 페이지에 근거(REQ/FN ID) 명시 `[연계 모드 전용]`
- [ ] 공통 페이지 포함 (홈, 로그인, 검색, 에러)
- [ ] 주요 사용자 흐름 3개 이상 정의
- [ ] 각 흐름에 진입 경로 + 전환 목표 명시

## § 2. 사이트맵 및 페이지 인벤토리
- [ ] GNB 항목 7개 이하
- [ ] Depth 4 이상 최소화
- [ ] 모든 페이지에 고유 IA-P### ID 부여
- [ ] ID 순번 연속성 유지
- [ ] 페이지명 중복 없음
- [ ] 메뉴명 한글 2~4자 권장, 중의적 표현 금지
- [ ] 모든 페이지에 템플릿 유형 지정
- [ ] 모든 페이지에 상태값 지정
- [ ] URL 전체 고유성 (중복 없음)
- [ ] URL 영문 소문자 + 하이픈, 최대 3 세그먼트
- [ ] REQ의 FR-### ID 역추적 가능 `[연계 모드 전용]`
- [ ] FN의 FN-### ID 역추적 가능 `[연계 모드 전용]`

## § 3. 네비게이션
- [ ] 모든 네비게이션 항목에 IA-N### ID 부여
- [ ] GNB 반응형 동작 정의 (모바일/태블릿/데스크톱)
- [ ] LNB 현재 위치 표시 방법 정의
- [ ] Footer 법적 고지 링크 포함 (개인정보처리방침, 이용약관)
- [ ] 3depth 이상 브레드크럼 정의
- [ ] 유틸리티 네비게이션 로그인/비로그인 상태 정의

## § 4. 콘텐츠 인벤토리
- [ ] 모든 콘텐츠 블록에 IA-C### ID 부여
- [ ] 필수/선택 구분 명시
- [ ] 데이터 소스 매핑 완료 (정적/CMS/DB/API/UGC)
- [ ] 템플릿별 표준 콘텐츠 블록 반영

## § 5. 산출물 생성
- [ ] MD 문서 7개 섹션 완비
- [ ] HTML 시각화 6개 탭 구현
- [ ] HTML self-contained (외부 의존성 없음)
- [ ] 완료 리포트 통계 정확성 확인
