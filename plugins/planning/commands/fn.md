# 기능정의서 (FN) Generator

REQ 기반으로 상세 기능정의서를 생성합니다.

## 전제조건
- REQ (요구사항정의서) 완료 필수
- 요구사항 ID 매핑

## 기능정의서 구조

### 1. 기능 목록 대시보드
```
📊 기능 구현 현황
├── 총 기능 수: ___개
├── 완료: ___개 (__%）
├── 진행중: ___개 (__%）
└── 대기: ___개 (__%）
```

### 2. 기능 상세 정의
```
┌─────────────────────────────────────┐
│ FN-001: [기능명]                      │
├─────────────────────────────────────┤
│ 연관 요구사항: FR-001, FR-002         │
│ 우선순위: High                        │
│ 담당: [개발자]                        │
├─────────────────────────────────────┤
│ 📝 기능 설명                          │
│ - 목적: [기능의 목적]                  │
│ - 트리거: [기능 실행 조건]             │
│ - 입력: [입력 데이터]                  │
│ - 처리: [처리 로직]                   │
│ - 출력: [출력 결과]                   │
├─────────────────────────────────────┤
│ ⚙️ 기술 명세                          │
│ - API: [엔드포인트]                   │
│ - DB: [테이블/컬럼]                   │
│ - 외부연동: [연동 시스템]              │
├─────────────────────────────────────┤
│ ✅ 검증 기준                          │
│ - 정상: [정상 케이스]                  │
│ - 예외: [예외 케이스]                  │
│ - 에러: [에러 처리]                   │
└─────────────────────────────────────┘
```

### 3. 기능 연관도
- 기능 간 의존 관계
- 데이터 흐름도
- 시스템 연동 구조

## 출력 형식
- HTML 탭 기반 문서
- 기능별 상세 카드
- 개발자 즉시 구현 가능
