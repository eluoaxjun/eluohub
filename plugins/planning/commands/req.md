# 요구사항정의서 (REQ) Generator

QST 기반으로 요구사항정의서를 생성합니다.

## 전제조건
- QST (고객질의서) 완료 필수
- QST 답변 내용 자동 반영

## REQ 대시보드 구조

### 1. 프로젝트 개요
```
📋 프로젝트 요구사항 대시보드
├── 작성일: [자동]
├── QST 기반 자동 생성
├── 실시간 추적 활성
└── 전체 진행률: 0%
```

### 2. 기능 요구사항 (우선순위별)
```
🔴 High Priority (필수 구현)
├── FR-001: [기능명] - 상태: 대기중 (0%)
├── FR-002: [기능명] - 상태: 대기중 (0%)
└── ...

🟡 Medium Priority (권장 구현)
├── FR-010: [기능명] - 상태: 대기중 (0%)
└── ...

🟢 Low Priority (선택 구현)
├── FR-020: [기능명] - 상태: 대기중 (0%)
└── ...
```

### 3. 비기능 요구사항
```
⚡ 성능: 페이지 로딩 ___초 이내
🔒 보안: [인증/암호화 요구사항]
📱 호환성: [브라우저/디바이스 지원]
♿ 접근성: WCAG 2.1 준수
```

### 4. 검증 기준
각 요구사항별:
- 완료 조건 (Definition of Done)
- 테스트 시나리오
- 담당자 및 일정

## 출력 형식
- HTML 대시보드 형태
- 진행률 실시간 추적 가능
- 개발팀 즉시 착수 가능
