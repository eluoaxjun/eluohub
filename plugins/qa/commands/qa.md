# 테스트계획서 (QA) Generator

프로젝트 테스트 계획을 생성합니다.

## 테스트 계획 구조

### 1. 테스트 범위
```
📋 테스트 대상
├── 기능 테스트: FN-001 ~ FN-0XX
├── 화면 테스트: UI-001 ~ UI-0XX
├── 성능 테스트: 응답속도, 부하
└── 보안 테스트: 인증, 권한, 취약점
```

### 2. 테스트 케이스
```
┌─────────────────────────────────────┐
│ TC-001: [테스트 케이스명]             │
├─────────────────────────────────────┤
│ 연관 기능: FN-001                    │
│ 우선순위: High                       │
│ 테스트 유형: 기능 테스트              │
├─────────────────────────────────────┤
│ 📝 테스트 시나리오                    │
│ 1. 사전조건: [조건]                   │
│ 2. 테스트 단계:                       │
│    Step 1: [행동]                    │
│    Step 2: [행동]                    │
│ 3. 예상 결과: [결과]                  │
├─────────────────────────────────────┤
│ ✅ 테스트 결과                        │
│ - 상태: Pass / Fail / Block          │
│ - 실행일: ____년 __월 __일            │
│ - 담당자: [이름]                      │
│ - 비고: [메모]                        │
└─────────────────────────────────────┘
```

### 3. 테스트 일정
```
📅 테스트 단계별 일정
├── 단위 테스트: __월 __일 ~ __월 __일
├── 통합 테스트: __월 __일 ~ __월 __일
├── 시스템 테스트: __월 __일 ~ __월 __일
├── UAT (사용자 수용 테스트): __월 __일 ~ __월 __일
└── 회귀 테스트: __월 __일 ~ __월 __일
```

### 4. 결함 관리
```
🐛 결함 심각도 분류
├── Critical: 서비스 불가
├── Major: 주요 기능 장애
├── Minor: 부분 기능 장애
└── Trivial: 경미한 이슈
```

### 5. 테스트 환경
- 개발 서버 (DEV)
- 스테이징 서버 (STG)
- 운영 서버 (PRD)

## 출력 형식
- 테스트 케이스 목록
- 테스트 결과 대시보드
- 결함 추적 현황
