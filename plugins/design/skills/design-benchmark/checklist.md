# Benchmark 체크리스트

## 대상 선정
- [ ] 동종 사이트 3개 이상
- [ ] 이종 사이트 1개 이상 (영감 소스)
- [ ] 어워드 수상작 1개 이상 포함 (GDWEB/Awwwards)

## 기본 정보
- [ ] 사이트명, URL, 업종, 분석일 기입
- [ ] 스크린샷 또는 참조 URL 첨부

## 디자인 분석
- [ ] 레이아웃 구조 (컬럼, 여백, 스크롤)
- [ ] 컬러 팔레트 추출 (메인/서브/액센트)
- [ ] 타이포그래피 확인 (폰트명, 사이즈 스케일)
- [ ] 이미지 스타일 (사진/일러스트/아이콘)

## UX/기능 분석
- [ ] 인터랙션 패턴 기록 (호버/스크롤/트랜지션)
- [ ] 반응형 대응 확인 (브레이크포인트)
- [ ] 주요 기능 목록 (검색/필터/다국어 등)

## 강점/약점
- [ ] 사이트별 강점 3개 + 약점 3개
- [ ] 차별화 기회 2개 이상

## 사용자 태깅
- [ ] AI 1차 분석 후 섹션별 사용자 판단 요청
- [ ] 채택/미채택 + 사유 기록
- [ ] 미완료 태깅 항목 0건

## 적용 제안
- [ ] 프로젝트에 적용할 포인트 명시
- [ ] 각 포인트에 출처 사이트 명시
- [ ] DK 수집 단계로 넘길 규칙 후보 식별

## 섹션별 벤치마크 매핑 (SectionBenchmark)
- [ ] 전체 섹션에 대한 레퍼런스 매핑 존재
- [ ] 각 섹션별 참조 레퍼런스 (순위 + 사이트명 + 적용 방식)
- [ ] 각 섹션별 디자인 타겟 (레이아웃/컬러/타이포/간격/호버)
- [ ] 각 섹션별 차별화 포인트 (복사가 아닌 변형)
- [ ] 시각 리듬 요약 표 포함 (섹션/패턴/배경/무게 4열)
- [ ] 연속 동일 패턴 0건 확인
- [ ] SectionBenchmark 파일 생성 또는 Benchmark 내 섹션 6에 포함
