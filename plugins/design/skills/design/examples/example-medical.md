## 브랜드 시트 발췌

톤: **신뢰(Trust), 전문(Professional), 따뜻함(Warm), 청결(Clean)** | 타겟: 환자 + 보호자

### 섹션 3: 컴포넌트 규칙

| 컴포넌트 | 규칙 |
|---------|------|
| 버튼(Primary) | bg #0066B3 / text white / radius 8px / 48px 높이 |
| 버튼(Emergency) | bg #DC2626 / text white — 응급실/긴급 전용 |
| 버튼(Secondary) | border 1px #0066B3 / text #0066B3 / bg white |
| 카드(진료과) | 아이콘(48px) + 진료과명 + 관련질환 태그 / radius 12px / hover border-color #0066B3 |
| 카드(의료진) | 프로필 사진(1:1 원형) + 이름 + 전문분야 + 예약 버튼 |
| 카드(건강정보) | 이미지 16:9 + 카테고리 태그 + 제목 + 발행일 |
| 검색 바 | height 56px / radius 28px / placeholder "증상, 진료과, 의료진 검색" |
| 탭 메뉴 | underline active 스타일 / #0066B3 active color |
| 스텝 인디케이터 | 원형 번호(1→2→3) / 활성=#0066B3 비활성=#D1D5DB |

### 섹션 5: Do's & Don'ts
- Do: 예약 CTA를 GNB + Hero + 진료과 상세 3곳에 배치 — 예약 전환 극대화
- Do: 진료시간/휴진 정보 실시간 반영 — 환자 혼란 방지
- Do: 모바일에서 전화 걸기(tel:) 즉시 접근 — 긴급 상황 대응
- Do: 의료진 프로필 사진 통일 (배경 동일, 정면, 상반신)
- Don't: 자극적 의료 이미지 사용 — 환자 불안 유발
- Don't: 진료과 목록에 축약어만 표시 — 환자 이해 불가
- Don't: 3단 이상 중첩 네비게이션 — 고령 환자 이탈
- Don't: 예약 폼 한 페이지에 10개+ 필드 — 단계별 분리 필수

## UI 발췌: 진료 예약 (UI-005)

| 항목 | 내용 |
|------|------|
| IA 페이지 | IA-P301 (진료 예약) |
| FN 매핑 | FN-005 (예약 등록), FN-006 (예약 조회) |
| Layout | Layout-005 참조 |

**UI 요소 — 3단계 스텝 폼**

| 단계 | 요소 | 상세 |
|------|------|------|
| Step 1 | 진료과 선택 | 카드 그리드 (아이콘+진료과명) / 검색 가능 |
| Step 1 | 의료진 선택 | 프로필 카드 (사진+이름+전문분야+가능일) |
| Step 2 | 날짜/시간 선택 | 캘린더 위젯 + 오전/오후 시간 슬롯 |
| Step 2 | 증상 입력 | Textarea (선택사항) |
| Step 3 | 확인/결제 | 예약 요약 테이블 + 동의 체크박스 + 확정 버튼 |

**스텝 인디케이터**

```
  ① 진료과 선택  ──  ② 날짜/시간  ──  ③ 예약 확인
  [활성]              [비활성]          [비활성]
```

**상태별 화면**

| 상태 | 표시 |
|------|------|
| Default | Step 1 (진료과 선택 화면) |
| Step 전환 | 스텝 인디케이터 업데이트 + fade 전환 |
| 예약 완료 | 체크 아이콘 + 예약번호 + 일시 요약 + [예약 확인서] 버튼 |
| 예약 불가 | "해당 일시는 예약이 마감되었습니다" + 다른 시간 제안 |
| Error | "예약 처리 중 오류가 발생했습니다" + 재시도 + 전화 예약 안내 |
