# 고객질의서 (plan-qst)

## 개요

고객 요구사항 파악을 위한 구조화된 질의서를 생성합니다.
비즈니스 목표, KPI, 팀 구성, 예산/일정 등을 선택형 질의로 파악합니다.

## 사전 준비

1. Claude Code CLI 설치 (v2.1 이상)
2. 이 폴더에서 Claude Code 실행

## 폴더 구조

```
plan-qst/                    ← 이 폴더에서 Claude Code 실행
├── .claude/                 ← 스킬 설정 (자동 인식, 수정 불필요)
├── README.md
├── input/                   ← 참고 자료 (제안서, 시안, RFP 등)
└── output/                  ← 산출물이 여기에 생성됩니다
```

## 사용 방법

### 자연어 실행

> "강남구 관광 홈페이지의 고객질의서를 만들어줘"
> "클라이언트에게 확인할 질문 목록 정리해줘"

### 슬래시 커맨드

> /plan-qst [프로젝트명 또는 요구사항 텍스트]

## 실행 모드

### 독립 모드
- 선행 산출물 없이 프롬프트만으로 실행
- 예: "이커머스 쇼핑몰 프로젝트의 고객질의서 만들어줘"

### 연계 모드
- 산출물 디렉토리에 다른 산출물이 있으면 보조 참조로 활용
- QST는 파이프라인 최상위이므로 직접 선행 산출물은 없음

## 산출물

- **파일명**: `QST_{프로젝트코드}_{버전}.md`
- **저장 경로**: `output/{프로젝트명}/{YYYYMMDD}/`
- **포함 내용**: 프로젝트 개요, 비즈니스 목표, 기능 요구사항, 기술 환경, 일정/예산 등 구조화 질의

## 품질 검증

- **Self-Check**: 스킬 완료 시 자동 실행 (3항목)
- **Reviewer**: planning-reviewer가 unit 모드로 Critical P/F 판정

## 다른 스킬과의 관계

| 관계 | 스킬 | 설명 |
|------|------|------|
| 선행 | 없음 | 파이프라인 최상위 |
| 후행 | plan-req | QST 답변 → REQ의 FR 도출 근거 |
