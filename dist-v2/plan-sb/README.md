# 화면설계서 (plan-sb)

## 개요

JSON 데이터를 입력받아 HTML/PDF 화면설계서를 자동 생성합니다.
테마 프리셋, v1/v2 스키마 자동 정규화, 다중 프레임 타입(Design, Description, MSG Case, Component Guide)을 지원합니다.

## 사전 준비

1. Claude Code CLI 설치 (v2.1 이상)
2. 이 폴더에서 Claude Code 실행

## 폴더 구조

```
plan-sb/                     ← 이 폴더에서 Claude Code 실행
├── .claude/                 ← 스킬 설정 (자동 인식, 수정 불필요)
├── README.md
├── input/                   ← 참고 자료 (JSON 데이터, 시안 등)
└── output/                  ← 산출물이 여기에 생성됩니다
```

## 사용 방법

### 자연어 실행

> "이 프로젝트의 화면설계서를 만들어줘"
> "JSON 데이터로 화면설계서를 생성해줘"

### 슬래시 커맨드

> /plan-sb [JSON 데이터 파일경로 또는 프로젝트 요구사항]

## 실행 모드

### 독립 모드
- 선행 산출물 없이 JSON 데이터 또는 프롬프트만으로 실행
- 예: "메인 페이지 화면설계서를 만들어줘"

### 연계 모드
- 산출물 디렉토리에 `FN_*.md` 존재 시 자동 활성화
- FN 기반 screens 자동 구성, REQ/IA 보조 참조

## 산출물

- **파일명**: `SB_{프로젝트코드}_{버전}.html`
- **저장 경로**: `output/{프로젝트명}/{YYYYMMDD}/`
- **포함 내용**: 페이지별 레이아웃, 구성 요소 정의, 인터랙션 명세, 반응형 브레이크포인트

## 품질 검증

- **Self-Check**: 스킬 완료 시 자동 실행
- **Reviewer**: planning-reviewer가 unit 모드로 Critical P/F 판정

## 다른 스킬과의 관계

| 관계 | 스킬 | 설명 |
|------|------|------|
| 선행 | plan-fn | FN → 화면 구성 요소 도출 (없으면 독립 모드) |
| 선행 | plan-ia | IA → 페이지 구조·네비게이션 참조 |
| 후행 | design | SB → 디자인 시안의 근거 |
