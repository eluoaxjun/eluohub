# 요구사항정의서 (plan-req)

## 개요

QST 기반으로 기능/비기능 요구사항을 정의합니다.
AC(인수조건), NFR 측정기준, 비즈니스 규칙, 추적성 매트릭스를 포함합니다.

## 사전 준비

1. Claude Code CLI 설치 (v2.1 이상)
2. 이 폴더에서 Claude Code 실행

## 폴더 구조

```
plan-req/                    ← 이 폴더에서 Claude Code 실행
├── .claude/                 ← 스킬 설정 (자동 인식, 수정 불필요)
├── README.md
├── input/                   ← 참고 자료 (제안서, 시안, RFP 등)
└── output/                  ← 산출물이 여기에 생성됩니다
```

## 사용 방법

### 자연어 실행

> "이 프로젝트의 요구사항을 정리해줘"
> "기능 요구사항과 비기능 요구사항을 정의해줘"

### 슬래시 커맨드

> /plan-req [QST 파일경로 또는 요구사항 텍스트]

## 실행 모드

### 독립 모드
- 선행 산출물 없이 프롬프트만으로 실행
- 예: "관광 홈페이지의 요구사항을 정의해줘"

### 연계 모드
- 산출물 디렉토리에 `QST_*.md` 존재 시 자동 활성화
- QST 답변에서 FR을 도출하고 Q→FR 추적성 매트릭스를 생성

## 산출물

- **파일명**: `REQ_{프로젝트코드}_{버전}.md`
- **저장 경로**: `output/{프로젝트명}/{YYYYMMDD}/`
- **포함 내용**: 비즈니스 목표, FR(기능 요구사항), NFR(비기능 요구사항), AC(인수조건), 추적성 매트릭스

## 품질 검증

- **Self-Check**: 스킬 완료 시 자동 실행 (9항목 — AC 금지 표현, ID 연속성, MoSCoW 분류 등)
- **Reviewer**: planning-reviewer가 unit 모드로 Critical P/F 판정

## 다른 스킬과의 관계

| 관계 | 스킬 | 설명 |
|------|------|------|
| 선행 | plan-qst | QST 답변 → FR 도출 근거 (없으면 독립 모드) |
| 후행 | plan-fn | FR → FN 매핑의 기반 |
| 후행 | plan-ia | FR → 페이지 매핑 참조 |
